# Отчеты о дефектах (Bug Reports)

В данном документе зафиксированы баги, обнаруженные в процессе тестирования.

---

### BUG-001: Итоговая сумма в корзине не пересчитывается после удаления товара
- **Статус:** Открыт (Open)
- **Приоритет:** Высокий (High)
- **Серьезность:** Критическая (Critical)
- **Окружение:** Chrome 122.0.6261, Windows 11

**Шаги воспроизведения:**
1. Добавить два товара в корзину.
2. Перейти в корзину.
3. Удалить один из товаров, нажав "Remove".

**Ожидаемый результат:** Товар удален, общая стоимость (Total) автоматически уменьшилась на цену удаленного товара.
**Фактический результат:** Товар визуально удален из списка, но поле "Total" по-прежнему показывает старую сумму за два товара.

### BUG-002: Перекрытие текста кнопкой "Зарегистрироваться" в мобильной версии
- **Статус:** Новый (New)
- **Приоритет:** Низкий (Low)
- **Серьезность:** Минорная (Minor) — UI дефект
- **Окружение:** iPhone 13, Safari (разрешение 390x844)

**Шаги воспроизведения:**
1. Открыть страницу регистрации на мобильном устройстве.
2. Прокрутить страницу до конца вниз.

**Ожидаемый результат:** Кнопка регистрации находится под текстом "Политика конфиденциальности", текст читаем.
**Фактический результат:** Кнопка накладывается (overlap) на текст политики, закрывая две нижние строки.

---

### BUG-003: Отсутствие валидации минимальной длины пароля при регистрации
- **Статус:** Открыт (Open)
- **Приоритет:** Средний (Medium)
- **Серьезность:** Значительная (Major) — Безопасность
- **Окружение:** Chrome, MacOS Sonoma

**Шаги воспроизведения:**
1. Начать регистрацию.
2. В поле "Пароль" ввести всего один символ "1".
3. Нажать кнопку регистрации.

**Ожидаемый результат:** Система выдает ошибку: "Пароль должен содержать от 6 до 20 символов". Регистрация невозможна.
**Фактический результат:** Система успешно создает аккаунт с паролем из одного символа. Ошибка валидации отсутствует.

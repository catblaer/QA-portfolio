# Чек-лист тестирования интернет-магазина (Sample E-commerce)

Данный документ содержит список проверок для быстрого регрессионного тестирования основных модулей системы.

---

## 1. Регистрация (Registration)
- [x] Регистрация с валидными данными (Email, Пароль > 6 симв.).
- [x] Попытка регистрации с уже существующим Email.
- [ ] Валидация пустого поля "Имя".
- [x] Проверка маски поля Email (наличие символа @ и точки).
- [ ] Отображение ошибки при вводе слишком короткого пароля (< 6 симв.).
- [x] Работа ссылки "Уже зарегистрированы? Войти".

---

## 2. Авторизация (Login)
- [x] Вход с корректным логином и паролем.
- [x] Вход с неверным паролем.
- [ ] Вход с несуществующим Email.
- [x] Проверка чувствительности к регистру в поле пароля (Case Sensitivity).
- [x] Функция "Выйти" (Logout) — проверка завершения сессии.
- [ ] Восстановление пароля через "Forgot Password" (если применимо).

---

## 3. Корзина покупок (Shopping Cart)
- [x] Добавление одного товара из каталога.
- [x] Добавление нескольких одинаковых товаров (увеличение счетчика).
- [x] Удаление товара из корзины кнопкой "Remove".
- [x] Автоматический пересчет итоговой суммы при изменении количества.
- [ ] Сохранение товаров в корзине после обновления страницы (F5).
- [ ] Переход к оформлению заказа (Checkout) из корзины.

---

## 4. Интерфейс и Адаптивность (UI/UX)
- [x] Корректное отображение на десктопных разрешениях (1920x1080).
- [x] Адаптивность верстки под мобильные устройства (iPhone/Android).
- [ ] Кликабельность логотипа (возврат на главную страницу).
- [x] Наличие футера со ссылками на социальные сети.
- [ ] Проверка 404 страницы при вводе несуществующего URL.

---

Легенда:
- [x] — Проверка пройдена успешно.
- [ ] — Проверка еще не выполнялась или ожидает исправления багов.
